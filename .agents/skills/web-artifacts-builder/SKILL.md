# TokenTriage Web Artifacts Builder

## Overview

TokenTriage serves three categories of web artifacts, all as **self-contained
single-file HTML documents** (inline CSS + JS, zero external build tools):

| Artifact | Path | Served at | Purpose |
|---|---|---|---|
| Chat Playground | `src/tokentriage/proxy/chat.html` | `/` and `/chat` | Interactive routing playground with SSE streaming |
| Live Dashboard | `src/tokentriage/proxy/dashboard.html` | `/dashboard` | Real-time stats from `/api/stats` (polled) |
| Evidence Dashboard | Generated by `src/tokentriage/evidence.py` → `_render_dashboard()` | Offline file | Standalone judge-facing report with embedded JSON data |

## Design System — Mandatory Tokens

All web artifacts **must** use the shared CSS custom-property palette. Never
hard-code colours outside `:root`.

### Tier colour mapping

Use these semantic colours when rendering per-tier indicators
(badges, progress bars, dots):

| Token | Tier | Meaning |
|---|---|---|
| `--cache` | T0 | Cache hit (green) |
| `--qwen` | T1 | Cheapest model (blue) |
| `--qwen-2` | T2 | Mid-tier (purple) |
| `--pro` | T3 | Largest / frontier (red) |
| `--verify` | — | Verifier / escalation (amber) |

### Typography

- **Body**: `14–15px`, `line-height 1.45`, `var(--sans)`.
- **Mono / labels**: `var(--mono)`, upper-case labels at `10–11px`,
  `font-weight 800`.
- **Headings**: `h1` at `17–28px` depending on context; no margins above,
  tight line-height.

### Card / panel pattern

```css
.card {
  background: var(--panel);
  border: 1px solid var(--rule);
  border-radius: 10px;
  box-shadow: var(--shadow);
  padding: 18px;
}
```

For semi-transparent overlays, use
`color-mix(in srgb, var(--panel) 94%, transparent)`.

## Architecture Rules

1. **Single-file HTML** — each artifact is one `.html` file containing all
   `<style>` and `<script>` inline. No external CSS/JS bundles, no npm.

2. **No external CDN dependencies** — everything must work offline.
   No Google Fonts links, no CDN-hosted libraries. System font stacks only.

3. **FastAPI serves HTML** — new pages are served via a route in
   `src/tokentriage/proxy/app.py` that reads the HTML file:
   ```python
   @app.get("/mypage", response_class=HTMLResponse)
   def mypage():
       html = Path(__file__).parent / "mypage.html"
       return html.read_text()
   ```

4. **Evidence dashboards are Python f-strings** — the offline evidence
   dashboard is generated by `evidence.py:_render_dashboard()` which embeds
   JSON data directly into the HTML via f-string interpolation. Use double
   braces `{{` / `}}` for literal JS braces inside f-strings.

5. **API contract**:
   - `/v1/route/stream` — SSE stream of routing-pipeline stages
     (the chat playground consumes this)
   - `/api/stats` — JSON stats for the live dashboard (polled every 2s)
   - `/conversations` — CRUD for saved chat sessions
   - New API endpoints go in `app.py`; keep the route handler thin and
     delegate logic to the appropriate module.

6. **Images** — static images live in `imgs/` at the repo root and are
   mounted at `/imgs` by FastAPI. Reference as `/imgs/filename.png`.

## Checklist for New Pages

- [ ] Create `src/tokentriage/proxy/<pagename>.html`
- [ ] Include `:root` light + dark tokens from the design system above
- [ ] Use the notebook-grid body background
- [ ] Add a FastAPI route in `app.py` using `HTMLResponse`
- [ ] Use `var(--*)` tokens everywhere — no hard-coded colours
- [ ] Test in both light and dark mode
- [ ] Add responsive breakpoints (`@media (max-width: 760px)`)

## Checklist for Modifying Existing Pages

- [ ] Preserve all existing CSS custom properties
- [ ] Keep the single-file structure — don't extract to separate files
- [ ] Test SSE streaming in chat.html still works after changes
- [ ] Test dashboard polling still functions
- [ ] Verify dark mode appearance
- [ ] Verify mobile / responsive layout

## Common Patterns

### SSE streaming (chat playground pattern)

```javascript
const resp = await fetch('/v1/route/stream', {
  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({ messages })
});
const reader = resp.body.getReader();
const decoder = new TextDecoder();
let buf = '';
while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  buf += decoder.decode(value, { stream: true });
  for (const line of buf.split('\n')) {
    if (line.startsWith('data: ')) {
      const evt = JSON.parse(line.slice(6));
      // Handle evt.stage: SECURITY, CACHE_CHECK, TRIAGE, POLICY,
      // DISPATCH, VERIFY, RESULT, QUARANTINE, ERROR
    }
  }
}
```

### Dashboard polling pattern

```javascript
async function poll() {
  const resp = await fetch('/api/stats?window_hours=24');
  const data = await resp.json();
  // Update DOM with data
}
setInterval(poll, 2000);
poll();
```
